## 1. プロジェクト概要

### 1.1 目的

カンバン方式のタスク管理ボードを提供し、タスクの状態を視覚的に整理・管理できるようにする。

### 1.2 対象ユーザー

個人でタスクを管理したいユーザー。

### 1.3 利用環境

- Web ブラウザ（デスクトップ）
- データはブラウザのローカルストレージに保存（サーバー不要）

---

## 2. 前提条件・制約・リスク

### 2.1 前提条件

| # | 内容 |
|---|------|
| P-1 | ユーザーはモダンブラウザ（Chrome / Safari / Firefox / Edge 最新版）を使用する |
| P-2 | JavaScript が有効であること |
| P-3 | インターネット接続は初回アクセス時のみ必要（以降はオフラインでも動作可能） |

### 2.2 制約事項

| # | 内容 |
|---|------|
| C-1 | データはブラウザの localStorage にのみ保存される（サーバーへの同期なし） |
| C-2 | 複数デバイス・複数ブラウザ間のデータ共有は非対応 |
| C-3 | ユーザー認証機能は非対応（1ブラウザ = 1ユーザー想定） |
| C-4 | 列の並び替え（ドラッグ）は非対応（本バージョンのスコープ外） |

### 2.3 リスクと対応策

| # | リスク | 影響度 | 対応策 |
|---|--------|--------|--------|
| R-1 | ブラウザのキャッシュ・履歴削除によるデータ消失 | 高 | ユーザーに事前周知。エクスポート機能は将来対応候補 |
| R-2 | localStorage の JSON データが破損している場合 | 中 | JSON パースエラー時はデフォルトデータで自動復旧 |
| R-3 | localStorage の容量上限（通常 5MB）超過 | 低 | タスク件数が極端に多くない限り問題なし |
| R-4 | ブラウザの localStorage が無効に設定されている場合 | 低 | データが保存されない旨をユーザーに説明 |

---

## 3. 非機能要件

### 3.1 パフォーマンス

| # | 要件 |
|---|------|
| Pf-1 | 初回ページ読み込みは通常のブロードバンド環境で 3 秒以内 |
| Pf-2 | カード追加・削除・移動はユーザー操作後に即座に画面へ反映され、目立ったラグが生じないこと |

### 3.2 対応ブラウザ

| ブラウザ | サポート |
|---------|---------|
| Google Chrome（最新版） | 対応 |
| Safari（最新版） | 対応 |
| Firefox（最新版） | 対応 |
| Microsoft Edge（最新版） | 対応 |
| Internet Explorer（全バージョン） | 非対応 |

---

## 4. スコープ外（将来の拡張候補）

| 機能 | 参考優先度 |
|------|----------|
| データのエクスポート / インポート | 高（R-1 のリスク軽減） |
| ユーザー認証・マルチユーザー対応 | 高 |
| サーバーへのデータ同期 | 高 |
| 列の並び替え（ドラッグ） | 中 |
| 期限日・優先度の設定 | 中 |
| タスクへのラベル・タグ付け | 中 |
| 検索・フィルタリング | 低 |
| ファイル添付 | 低 |
| コメント機能 | 低 |
| 通知機能 | 低 |
