## 1. プロジェクト概要

### 1.1 目的

カンバン方式のタスク管理ボードを提供し、タスクの状態を視覚的に整理・管理できるようにする。

### 1.2 対象ユーザー

個人でタスクを管理したいユーザー。

### 1.3 利用環境

- Web ブラウザ（デスクトップ）
- データはサーバーサイド（PostgreSQL）に保存。バックエンドサーバーへの常時接続が必要

---

## 2. 前提条件・制約・リスク

### 2.1 前提条件

| # | 内容 |
|---|------|
| P-1 | ユーザーはモダンブラウザ（Chrome / Safari / Firefox / Edge 最新版）を使用する |
| P-2 | JavaScript が有効であること |
| P-3 | バックエンドサーバー（Spring Boot）への常時ネットワーク接続が必要。オフライン動作は非対応 |

### 2.2 制約事項

| # | 内容 |
|---|------|
| C-1 | データは PostgreSQL に保存される。複数デバイス・複数ブラウザ間でデータは共有される |
| C-2 | ユーザー認証機能は非対応（シングルユーザー想定。全アクセスが同一データを参照する） |
| C-3 | 列の並び替え（ドラッグ）は非対応（本バージョンのスコープ外） |

### 2.3 リスクと対応策

| # | リスク | 影響度 | 対応策 |
|---|--------|--------|--------|
| R-1 | バックエンドサーバー障害による操作不能 | 高 | フロントエンドでエラーメッセージを表示し、リトライ案内をする |
| R-2 | ネットワーク遅延による操作レスポンス低下 | 中 | ローディングインジケーターを表示し、操作中であることを明示する |
| R-3 | PostgreSQL のデータ破損・消失 | 高 | 定期バックアップを運用ルールとして設定する |

## 3. 非機能要件

### 3.1 パフォーマンス

| # | 要件 |
|---|------|
| Pf-1 | 初回ページ読み込みは通常のブロードバンド環境で 3 秒以内 |
| Pf-2 | カード追加・削除・移動はユーザー操作後に即座に画面へ反映され、目立ったラグが生じないこと |

### 3.2 対応ブラウザ

| ブラウザ | サポート |
|---------|---------|
| Google Chrome（最新版） | 対応 |
| Safari（最新版） | 対応 |
| Firefox（最新版） | 対応 |
| Microsoft Edge（最新版） | 対応 |
| Internet Explorer（全バージョン） | 非対応 |

## 4. スコープ外（将来の拡張候補）

| 機能 | 参考優先度 |
|------|----------|
| ユーザー認証・マルチユーザー対応 | 高 |
| 列の並び替え（ドラッグ） | 中 |
| 期限日・優先度の設定 | 中 |
| タスクへのラベル・タグ付け | 中 |
| 検索・フィルタリング | 低 |
| データのエクスポート / インポート | 低 |
| ファイル添付 | 低 |
| コメント機能 | 低 |
| 通知機能 | 低 |
